# Data directory

Each dataset is a subdirectory holding the CSVs described in
[`../DATA_SCHEMA.md`](../DATA_SCHEMA.md). The analysis selects one with
`--dataset <name>`, which resolves to `analysis/data/<name>/`.

```
analysis/data/
  synthetic/        # generated by `python cli.py synth`; gitignored, never released
  iclr2026/         # real export: submissions.csv, gradings.csv [, reviews.csv]
  ...
```

## Adding a real dataset

1. Produce `submissions.csv` and `gradings.csv` (and optionally `reviews.csv`)
   conforming to `../DATA_SCHEMA.md`. See the repo README, "Producing the real
   data", for the reference exporter.
2. Drop them in `analysis/data/<name>/`.
3. Run `python cli.py all --dataset <name>`.

The schema is validated on load (`schema.py::validate`); a malformed export
raises before any figure renders.

## What is and isn't committed

- **Committed** (the public release unit): the de-identified `submissions.csv`,
  `gradings.csv`, and `reviews.csv` for real datasets — their contents
  (forum ids, public decisions, public reviewer ratings) are already public on
  OpenReview.
- **Never committed** (`.gitignore`): the `synthetic/` directory, any
  `findings.jsonl` (AIPR-generated weaknesses about named papers), and any
  manuscript `*.pdf`.
